01. About Me

A philosophy rooted in execution discipline.

I'm a registered nurse who became a prior authorization supervisor, then started writing code to eliminate the inefficiencies I was managing.

That path gave me something uncommon: I understand clinical workflows because I've worked them. I understand payer operations because I supervise them. And I understand how to build software tools because I got tired of waiting for someone else to do it.

At TMHP, I supervise PA clinicians processing Texas Medicaid authorizations — managing caseloads, building evaluation frameworks, handling complex eligibility cases across MCO transitions. Along the way, I built a Python GUI that 5x'd my own PA processing speed, an inventory bot that eliminated an hour of daily supervisor busywork, a Flask dashboard with a greedy algorithm that optimizes daily staffing assignments, and a Copilot RAG agent that gives citation-backed answers from the 900+ page Texas Medicaid manual.

I build before I'm asked. Every tool in my portfolio was self-initiated. No one assigned me to automate PA processing or index the TMPPM. I saw a problem, scoped a solution, and shipped it.

I think in systems, not tasks. My ADHD brain naturally maps process dependencies, bottlenecks, and cascading effects. I compensate for attention variability with structured frameworks — which means I build systems that work whether I'm having a sharp day or not. That discipline shows up in everything I create.

I translate across domains. I can explain a clinical policy to an engineer, walk a product team through payer operations workflows, and demo a Python tool to a clinical director — all in language that lands. This bridging capability is rare because most people live in only one of those worlds.
